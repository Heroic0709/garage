# 基于车牌识别的停车场管控系统 — 需求说明文档

## 一、项目概述

### 1.1 项目名称
基于车牌识别的停车场管控系统

### 1.2 项目简介
通过车牌识别技术实现停车场车辆的入场、计费、缴费、出场全流程自动化管理。系统采用 B/S 架构，前端通过上传车牌图片模拟闸机摄像头采集，后端调用 OpenCV + PaddleOCR 完成车牌识别，并结合计费规则实现停车费用计算与缴费管理。

### 1.3 系统架构
```
┌─────────────────────────────────────────────────┐
│                   前端 (Vue3)                    │
│    入场页 │ 缴费页 │ 出场页 │ 管理后台           │
└──────────────────┬──────────────────────────────┘
                   │ HTTP/REST API (Axios)
┌──────────────────▼──────────────────────────────┐
│               后端 (FastAPI)                     │
│   /api/entry  /api/pay  /api/exit  /api/records │
│                    │                            │
│          ┌─────────▼─────────┐                  │
│          │  车牌识别引擎      │                  │
│          │  OpenCV+PaddleOCR │                  │
│          └───────────────────┘                  │
│                    │                            │
│          ┌─────────▼─────────┐                  │
│          │    MySQL 8.0      │                  │
│          └───────────────────┘                  │
└─────────────────────────────────────────────────┘
```

## 三、计费规则

| 规则项 | 说明 |
|--------|------|
| 计费方式 | 阶梯式：首小时 5 元，之后每小时 3 元 |
| 免费时段 | 无 |
| 不满 1 小时 | 按 1 小时计算（向上取整） |
| 封顶 | 无 |
| 车型 | 统一费率，不区分车型 |
| 缴费有效期 | 缴费成功后 **20 分钟** 内必须出场（可配置），超时缴费失效 |
| 缴费失效处理 | 方案 B：status 回退为 `parking`，出场时从 **原始入场时间** 重新计费，**抵扣已缴金额**，只补差额 |
| 计费公式 | `费用 = 5 + max(0, ceil(停车时长 - 1)) × 3` |

**示例：**

| 停车时长 | 计费时长 | 费用 |
|----------|----------|------|
| 30 分钟 | 1 小时 | 5 元 |
| 1 小时 | 1 小时 | 5 元 |
| 1 小时 10 分钟 | 2 小时 | 8 元 |
| 3 小时 | 3 小时 | 5 + 2×3 = 11 元 |
| 5 小时 30 分钟 | 6 小时 | 5 + 5×3 = 20 元 |

**缴费超时示例（方案 B）：**

| 时间线 | 事件 | 费用说明 |
|--------|------|----------|
| 10:00 | 入场 | — |
| 12:00 | 缴费 8 元（覆盖 10:00→12:00） | 已缴 8 元 |
| 12:30 | 出场（距缴费已过 30 分钟，超时失效） | 10:00→12:30 = 3 小时 = 11 元，抵扣 8 元，**补缴 3 元** |

## 四、数据库设计

### 4.1 停车记录表 `parking_record`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT AUTO_INCREMENT PK | 主键 |
| `plate_number` | VARCHAR(20) NOT NULL | 车牌号 |
| `plate_image_url` | VARCHAR(500) | 入场车牌图片路径 |
| `entry_time` | DATETIME NOT NULL | 入场时间 |
| `exit_time` | DATETIME | 出场时间（NULL 表示在场） |
| `status` | ENUM('parking','paid','exited') | 状态：在场/已缴费/已出场 |
| `created_at` | DATETIME DEFAULT NOW() | 记录创建时间 |

### 4.2 缴费记录表 `payment_record`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT AUTO_INCREMENT PK | 主键 |
| `parking_record_id` | INT FK | 关联停车记录 |
| `plate_number` | VARCHAR(20) NOT NULL | 车牌号（冗余，便于查询） |
| `amount` | DECIMAL(10,2) NOT NULL | 缴费金额 |
| `park_hours` | INT NOT NULL | 计费小时数 |
| `status` | ENUM('valid','expired') | 状态：valid-有效 / expired-已过期 |
| `pay_time` | DATETIME NOT NULL | 缴费时间 |

### 4.3 ER 关系

```
parking_record (1) ──→ payment_record (N)
     │                    一条停车记录可能对应多笔缴费
     │                    （正常情况一笔，支持补缴/过期重新缴费）
     │                    payment_record.status = 'expired' 表示该笔缴费已过期
```

## 五、业务流程

### 5.1 整体状态流转

```
                    ┌─────────────┐
                    │   上传入场图片  │
                    └──────┬──────┘
                           │ 车牌识别成功
                           ▼
                    ┌─────────────┐
                    │  parking     │ (在场)
                    │  自动计费开始  │
                    └──────┬──────┘
                           │ 用户主动缴费
                           ▼
                    ┌─────────────┐      超过20分钟未出场
                    │   paid       │ (已缴费) ──────────────┐
                    └──────┬──────┘                        │
                           │ 20分钟内上传出场图片            │
                           ▼                               │
           ┌───────────────────────────┐                   │
           │  出场检测                  │                   │
           │  pay_time + 20min > now?  │                   │
           └─────┬──────────┬──────────┘                   │
                 │ 是       │ 否（超时）                    │
                 ▼          ▼                              │
           ┌─────────┐  ┌──────────────────────────┐       │
           │ exited  │  │ 缴费失效，status 回退     │◄──────┘
           │ (已出场) │  │ payment_record→expired    │
           └─────────┘  │ 从原始 entry_time 重新计费 │
                        │ 抵扣已缴金额，补收差额      │
                        └─────────────┬────────────┘
                                      │ 用户重新缴费
                                      ▼
                               ┌─────────────┐
                               │   paid       │ (重新缴费)
                               └──────┬──────┘
                                      │ 20分钟内出场
                                      ▼
                               ┌─────────────┐
                               │   exited     │
                               └─────────────┘
```

### 5.2 入场流程

```
用户打开「入场」页面
  → 上传车牌图片
  → 后端调用车牌识别引擎
  → 校验该车牌是否已有 status='parking' 或 'paid' 的记录
     └─ 有 → 拒绝入场，提示"该车辆已在场内，不可重复入场"
  → 识别成功 → 创建 parking_record（status=parking）
  → 保存原始图片到服务器
  → 前端显示入场成功（车牌号 + 入场时间）
  
  识别失败 → 提示用户重新上传或手动输入车牌号
```

### 5.3 缴费流程

```
用户打开「缴费」页面
  → 输入车牌号并查询
  → 后端查询该车牌 status=parking 的最新记录
     ├─ 无记录 → 提示"未找到该车辆的停车记录"
     └─ 有记录 → 计算应缴金额并显示（入场时间、已停时长、应付金额）
  → 用户点击「确认缴费」
  → 生成 payment_record（status='valid'）
  → 更新 parking_record.status = 'paid'
  → 前端显示缴费成功 + "请在 20 分钟内出场"倒计时

缴费过期不主动检测，在出场环节触发判断。
```

### 5.4 出场流程

```
用户打开「出场」页面
  → 上传车牌图片
  → 后端调用车牌识别引擎
  → 查询该车牌最新记录
     ├─ 无记录 → 提示"未找到入场记录"
     ├─ status=parking → 提示"请先缴费"，引导跳转缴费页
     ├─ status=paid →
     │    ├─ pay_time + 20分钟 > 当前时间 → 在有效期内
     │    │    → 更新 exit_time + status='exited' → 放行
     │    └─ pay_time + 20分钟 <= 当前时间 → 缴费已过期
     │         → 将该笔 payment_record 标记为 'expired'
     │         → parking_record.status 回退为 'parking'
     │         → 从原始 entry_time 重新计算总费用
     │         → 抵扣该车牌所有历史 expired 缴费金额
     │         → 提示"缴费已过期，需补缴 X 元"，引导跳转缴费页
     └─ status=exited → 提示"该车辆已出场"
```

## 六、页面设计

- 「确认入场」按钮
- 入场记录展示区（最近入场车辆列表）

- 车牌号输入框 + 「查询」按钮
- 查询结果展示卡片：
  - 车牌号、入场时间、已停时长、应付金额

- **计费规则展示**：当前的费率说明

## 七、API 接口设计

### 7.1 车牌识别
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/recognize` | 上传图片，返回识别结果 |

**Response:**
```json
{
  "plate": "豫B·60P70",
  "confidence": 0.998,
  "success": true
}
```

### 7.2 入场
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/entry` | 上传图片并生成入场记录 |

**业务校验：** 若该车牌已有 `status='parking'` 或 `status='paid'` 的记录，返回 400 错误"该车辆已在场内"。

**Response（成功）:**
```json
{
  "id": 1,
  "plate_number": "豫B·60P70",
  "entry_time": "2026-06-24 18:00:00",
  "status": "parking"
}
```

### 7.3 缴费查询
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/payment/query?plate=豫B·60P70` | 查询待缴费信息 |

**Response:**
```json
{
  "plate_number": "豫B·60P70",
  "entry_time": "2026-06-24 10:00:00",
  "park_hours": 8,
  "amount": 26.00
}
```

### 7.4 确认缴费
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/payment/pay` | 执行缴费 |

**Request:**
```json
{ "plate_number": "豫B·60P70" }
```

**Response:**
```json
{
  "amount": 8.00,
  "park_hours": 2,
  "pay_time": "2026-06-24 12:00:00",
  "expire_minutes": 20,
  "expire_time": "2026-06-24 12:20:00",
  "message": "缴费成功，请在 20 分钟内出场"
}
```

### 7.5 出场校验
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/exit` | 上传图片，校验并出场 |

**Response（在有效期内，放行）:**
```json
{
  "plate_number": "豫B·60P70",
  "can_exit": true,
  "message": "已缴费，请通行"
}
```

**Response（缴费已过期，需补缴）:**
```json
{
  "plate_number": "豫B·60P70",
  "can_exit": false,
  "reason": "expired",
  "total_amount": 11.00,
  "paid_amount": 8.00,
  "need_pay": 3.00,
  "message": "缴费已过期，需补缴 3.00 元"
}
```

### 7.6 管理后台
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/records?page=1&status=parking` | 停车记录列表（分页+筛选） |
| GET | `/api/admin/statistics` | 收费统计数据（仅统计 status='valid' 的缴费） |

## 十一、Phase 2 功能规划（v2.0）

> 基于系统 v1.2 完成后的增强计划，按优先级分阶段实施。

### 11.1 用户登录与权限体系

**功能描述：** 引入 JWT 认证，区分管理员和普通用户角色。

| 角色 | 权限 |
|------|------|
| 管理员 (`admin`) | 管理后台（所有记录 + 统计）、黑名单管理、操作日志查看 |
| 普通用户 (`user`) | 缴费、查看自己的停车记录和缴费历史、会员注册 |

**认证方案：**
- 用户名 + 密码登录，密码 bcrypt 哈希存储
- JWT token 有效期 7 天，前端存储于 localStorage
- 后端通过 Depends 注入 `current_user`，按角色控制接口权限

**前端变更：**
- 落地页增加登录入口（或系统入口带登录拦截）
- MainLayout 顶部右侧展示当前用户头像/昵称
- 管理员可见管理后台菜单项

### 11.2 会员系统

**功能描述：** 普通用户注册会员（¥20/月），享受优惠停车费率 ¥3/小时。

| 项目 | 说明 |
|------|------|
| 会员费 | ¥20 / 月 |
| 会员费率 | ¥3/小时（不足 1 小时按 1 小时计算） |
| 非会员费率 | 首小时 ¥5 + 后续 ¥3/小时 |
| 有效期 | 购买日起 30 天 |
| 续费方式 | 手动续费，到期自动降为非会员 |
| 单次封顶 | 无封顶 |

**计费逻辑：** 查询费用时，根据用户会员状态判断费率：
- 会员：`费用 = ceil(停车时长) × 3`
- 非会员（原逻辑）：`费用 = 5 + max(0, ceil(停车时长 - 1)) × 3`

**需新增数据库对象：**
- `users` 表：增加 `membership_expire` 字段记录会员到期时间

### 11.3 黑名单功能

**功能描述：** 管理员可将指定车牌加入黑名单，黑名单车辆禁止入场。

| 项目 | 说明 |
|------|------|
| 操作权限 | 仅管理员 |
| 黑名单类型 | 永久 / 限时（N 天后自动解除） |
| 拦截时机 | 入场接口校验时（OCR 识别后、手动输入确认前） |
| 状态 | active（生效中）/ expired（已过期） |

**需新增数据库对象：**
- `blacklist` 表：plate_number, reason, black_type, expire_at, created_by, status, created_at

### 11.4 手机端支持（响应式适配）

**方案：** 前端做响应式适配，手机浏览器直接访问。

| 改动点 | 说明 |
|--------|------|
| 左侧导航栏 | 手机端（<768px）折叠为汉堡菜单 |
| 页面布局 | max-width 限制改为 100%，自适应屏幕宽度 |
| 车牌输入框 | 手机端缩小输入框尺寸，虚拟键盘适配 |
| 上传区 | 手机端可调用摄像头拍照上传 |

### 11.5 补充功能

#### 储值钱包
- 用户可预充值，出场时自动扣费
- 余额不足时提示充值，也可从入场计费开始（不影响入场）

#### 用户停车记录查询
- 普通用户登录后查看自己绑定车牌的历史记录
- 字段：入场/出场时间、费用、缴费状态

#### 车辆多绑定
- 一个用户可绑定多辆车（家庭多车场景），共享会员权益
- 非绑定车辆不可使用该用户账户缴费

#### 操作日志
- 管理员所有敏感操作（设黑名单、修改记录、退款等）记录到 `audit_log` 表
- 可追溯，含操作人、操作类型、操作时间、IP

#### 数据导出
- 管理后台支持将停车记录导出为 Excel（需后端新增 `openpyxl` 依赖）
