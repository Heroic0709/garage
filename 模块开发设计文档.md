## 一、开发总览

### 1.1 开发原则

| 原则 | 说明 |
|------|------|
| 后端优先 | 先完成数据模型和 API，前端再接入 |
| 自底向上 | 每个阶段按 模型 → 服务 → 路由 → 入口 的顺序 |
| 按功能模块 | 以业务功能为单位，一个模块完成后再进入下一个 |
| 即时验证 | 每完成一个模块，编写验证步骤确保可运行 |

### 1.2 模块划分与开发顺序

```
Phase 0: 项目骨架搭建
  ├── M0-1  创建项目目录结构
  └── M0-2  安装后端依赖

Phase 1: 后端基础设施
  ├── M1-1  [models.py]       数据库模型（parking_record / payment_record）
  ├── M1-2  [database.py]     数据库连接配置
  └── M1-3  [config.py]       系统配置（计费参数、有效期等）

Phase 2: 后端业务服务层
  ├── M2-1  [services/billing.py]      计费服务（核心计费公式 + 抵扣逻辑）
  ├── M2-2  [services/plate_service.py] 车牌识别服务（封装 main.py）
```

## 三、Phase 1: 后端基础设施

### M1-1 数据库模型 `backend/models.py`

**功能：** 使用 SQLAlchemy ORM 定义两张表

**表结构（对应现有数据库）：**

#### parking_record

| 列名 | 类型 | 约束 |
|------|------|------|
| `id` | Integer | PK, auto_increment |
| `plate_number` | String(20) | NOT NULL, index |
| `plate_image` | String(500) | nullable |
| `entry_time` | DateTime | NOT NULL, index |
| `exit_time` | DateTime | nullable |
| `status` | String(20) | NOT NULL, default='parking', index |
| `created_at` | DateTime | NOT NULL, default=func.now() |

#### payment_record

| 列名 | 类型 | 约束 |
|------|------|------|
| `id` | Integer | PK, auto_increment |
| `parking_record_id` | Integer | FK → parking_record.id |
| `plate_number` | String(20) | NOT NULL, index |
| `amount` | DECIMAL(10,2) | NOT NULL |
| `park_hours` | Integer | NOT NULL |
| `status` | String(20) | NOT NULL, default='valid' |
| `pay_time` | DateTime | NOT NULL, default=func.now() |

**关键设计点：**
- 使用 `declarative_base()` 定义模型
- `status` 字段由于 MySQL ENUM 在 SQLAlchemy 中的映射，使用 `String(20)` 类型，业务层确保值只能是 `parking`/`paid`/`exited` 或 `valid`/`expired`
- 两表之间定义 `relationship()`

**验证方式：**
- 脚本执行 `from backend.models import ParkingRecord, PaymentRecord` 无报错
- 手动用 SQL 查询 `mysql -u root -p123456 parking_system -e "SHOW TABLES"` 确认表存在

## 四、Phase 2: 后端业务服务层

### M2-1 计费服务 `backend/services/billing.py`

**功能：** 停车费用计算，包含核心计费和方案 B 过期抵扣逻辑

**核心方法：**

```
calculate_fee(entry_time: datetime, exit_time: Optional[datetime] = None) -> tuple:
    """
    计算停车费用
    参数:
        entry_time: 入场时间
        exit_time: 出场时间，默认 None 则用当前时间
    返回:
        (park_hours: int, amount: float)
        例: (3, 11.0) 表示 3小时共11元
    """

calculate_deduction(plate_number: str, session) -> float:
    """
    查询该车牌在当前 parking_record 下所有 expired 缴费的总金额
    用于方案B抵扣
    返回:
        已缴可抵扣总额
    """
```

**计算公式：**
```
park_hours = max(1, ceil(elapsed_hours))   # 不满1小时算1小时
if park_hours == 1:
    amount = 5
else:
    amount = 5 + (park_hours - 1) * 3
```

**验证方式：**
- `calculate_fee(datetime(2026,6,24,10,0), datetime(2026,6,24,10,30))` → `(1, 5.0)`
- `calculate_fee(datetime(2026,6,24,10,0), datetime(2026,6,24,12,10))` → `(3, 11.0)`
- `calculate_fee(datetime(2026,6,24,10,0), datetime(2026,6,24,15,0))` → `(5, 17.0)`

```
recognize_and_save(image_file: UploadFile) -> dict:
```

## 五、Phase 3: 后端 API 路由

### M3-1 入场接口 `backend/routes/entry.py`

**路由：** `POST /api/entry`

**处理流程：**

```
接收图片文件
  → 调用 plate_service.recognize_and_save()
    → 识别失败 → 返回 422 {success: false, error: "xxx"}
  → 查询该车牌是否有 status IN ('parking','paid') 的记录
    → 有 → 返回 400 {message: "该车辆已在场内，不可重复入场"}
  → 创建 parking_record
    plate_number = 识别结果
    plate_image = 保存的图片路径
    entry_time = datetime.now()
    status = 'parking'
  → 返回 EntryResponse
```

**关键设计点：**
- `plate_image` 存相对路径，从项目根算起，如 `backend/uploads/xxx.jpg`
- `entry_time` 在 Python 端生成，不使用 MySQL 的 NOW()，确保与计费时间一致

**验证方式：**
- Postman/curl 上传测试图片，返回 200 + 入场信息
- 再次上传同一张图片，返回 400 "该车辆已在场内"

---

### M3-2 缴费接口 `backend/routes/payment.py`

#### 3-2a 缴费查询 `GET /api/payment/query?plate=豫B·60P70`

**处理流程：**

```
接收 plate_number
  → 查询 status='parking' 的最新记录
    → 无 → 返回 404
    → 有 → 调用 billing.calculate_fee(entry_time)
           返回 PaymentQueryResponse
```

#### 3-2b 确认缴费 `POST /api/payment/pay`

**处理流程：**

```
接收 {plate_number}
  → 查询 status='parking' 的最新记录
    → 无 → 返回 404
  → 调用 billing.calculate_fee(entry_time) 得到 amount, park_hours
  → 创建 payment_record(status='valid')
  → 更新 parking_record.status = 'paid'
  → 返回 PayResponse（含 expire_time = pay_time + 20分钟）
```

**关键设计点：**
- 缴费金额**在服务端重新计算**（不要信任前端传来的金额）
- `expire_time` 是 `pay_time + timedelta(minutes=PAYMENT_EXPIRE_MINUTES)`

**验证方式：**
- GET 查询已有停车记录 → 返回金额正确
- POST 缴费 → status 变为 'paid'，payment_record 生成
- 缴费后再次查询 → 返回 404（因为 status 已是 'paid' 不是 'parking'）

### M3-3 出场接口 `backend/routes/exit.py`

**路由：** `POST /api/exit`

**处理流程（最复杂 — 方案B核心）：**

```
接收图片文件
  → 调用 plate_service.recognize_and_save()
    → 识别失败 → 返回 422
  → 查询该车牌最新记录（status IN ('parking','paid')）
    → 无 → 返回 {can_exit: false, reason: "no_record", message: "未找到入场记录"}
    → status='parking' → 返回 {can_exit: false, reason: "not_paid", message: "请先缴费"}
    → status='paid' →
         查询该 parking_record 下最新一笔 status='valid' 的 payment_record
            → 无 valid 记录 → 返回 {can_exit: false, reason: "not_paid"}
            → pay_time + expire_minutes > now() → 在有效期内
                → 更新 parking_record.exit_time = now(), status = 'exited'
                → 返回 {can_exit: true, message: "已缴费，请通行"}
            → pay_time + expire_minutes <= now() → 缴费过期！
                → 将 payment_record.status 改为 'expired'
                → parking_record.status 回退为 'parking'
                → 调用 billing.calculate_fee(entry_time) 计算新总费用
                → 调用 billing.calculate_deduction(plate_number) 获取可抵扣金额
                → need_pay = total_amount - paid_amount
                → 返回 {can_exit: false, reason: "expired",
                        total_amount, paid_amount, need_pay,
                        message: "缴费已过期，需补缴 X 元"}
```

**关键设计点：**
- 过期检测必须在出场时触发，不做定时任务
- 过期后将 `parking_record.status` 回退为 `parking`，使得缴费接口可以再次查询到
- 抵扣金额只包含当前 parking_record 下所有 expired 的缴费，不跨记录
- 先标记 expired 再回退 status，顺序不能错

**验证方式：**
- 缴费 + 立即出场 → 正常放行
- 缴费 + 等待 20 分钟 + 出场 → 返回过期，需要补缴
- 过期后重新缴费 + 出场 → 正常放行
- status='parking' 直接出场 → 提示先缴费

### M3-4 管理后台接口 `backend/routes/admin.py`

#### 3-4a 记录列表 `GET /api/admin/records?page=1&page_size=10&status=parking&plate=豫B`

**处理流程：**

```
接收 page, page_size, status(可选), plate(可选)
  → 构建查询
    parking_record LEFT JOIN payment_record (只取 status='valid' 的)
    支持按 status 筛选、按 plate_number 模糊搜索
    按 entry_time DESC 排序
    分页 offset = (page-1)*page_size
  → 返回 {total, page, page_size, items: [...]}
```

**每条 item 包含：** id, plate_number, entry_time, exit_time, status, paid_amount

#### 3-4b 收费统计 `GET /api/admin/statistics`

**返回：**

```
today_income:  当天所有 valid 缴费的 SUM(amount)
month_income:  当月所有 valid 缴费的 SUM(amount)
total_income:  历史所有 valid 缴费的 SUM(amount)
parked_count:  当前 status IN ('parking','paid') 的记录数
```

**关键设计点：**
- 统计只算 `status='valid'` 的缴费，expired 的不计入

**验证方式：**
- 调用接口，返回 JSON 含四项统计数字

```
create_access_token(user_id, role, username) -> str:
```
