## 三、表结构设计

### 3.1 停车记录表 `parking_record`

#### 表说明
记录车辆从入场到出场的完整生命周期，是系统的核心业务表。

#### 字段定义

| 序号 | 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|------|--------|------|------|--------|--------|------|
| 1 | `id` | INT | — | NOT NULL | AUTO_INCREMENT | 主键，自增 |
| 2 | `plate_number` | VARCHAR | 20 | NOT NULL | — | 车牌号（如"豫B·60P70"） |
| 3 | `plate_image` | VARCHAR | 500 | NULL | NULL | 入场车牌图片的存储路径 |
| 4 | `entry_time` | DATETIME | — | NOT NULL | — | 入场时间 |
| 5 | `exit_time` | DATETIME | — | NULL | NULL | 出场时间（在场时为空） |
| 6 | `status` | ENUM | — | NOT NULL | 'parking' | 状态：parking/paid/exited |
| 7 | `created_at` | DATETIME | — | NOT NULL | CURRENT_TIMESTAMP | 记录创建时间 |

#### 状态枚举说明

| 值 | 含义 | 说明 |
|-----|------|------|
| `parking` | 在场 | 已入场，尚未缴费 |
| `paid` | 已缴费 | 已缴费，等待出场 |
| `exited` | 已出场 | 已完成出场，记录归档 |

#### DDL

```sql
CREATE TABLE `parking_record` (
    `id`            INT             NOT NULL AUTO_INCREMENT,
    `plate_number`  VARCHAR(20)     NOT NULL COMMENT '车牌号',
    `plate_image`   VARCHAR(500)    NULL     COMMENT '入场车牌图片路径',
    `entry_time`    DATETIME        NOT NULL COMMENT '入场时间',
    `exit_time`     DATETIME        NULL     COMMENT '出场时间',
    `status`        ENUM('parking','paid','exited')
                                    NOT NULL DEFAULT 'parking' COMMENT '状态',
    `created_at`    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    INDEX `idx_plate_number` (`plate_number`),
    INDEX `idx_status` (`status`),
    INDEX `idx_entry_time` (`entry_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='停车记录表';
```

#### 索引说明

| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| `PRIMARY` | `id` | 主键 | 唯一标识 |
| `idx_plate_number` | `plate_number` | 普通索引 | 按车牌查询（缴费查询、出入场校验） |
| `idx_status` | `status` | 普通索引 | 按状态筛选（查询在场车辆） |
| `idx_entry_time` | `entry_time` | 普通索引 | 按时间排序和统计 |

### 3.2 缴费记录表 `payment_record`

#### 表说明
记录每一笔缴费明细，关联停车记录。

#### 字段定义

| 序号 | 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|------|--------|------|------|--------|--------|------|
| 1 | `id` | INT | — | NOT NULL | AUTO_INCREMENT | 主键，自增 |
| 2 | `parking_record_id` | INT | — | NOT NULL | — | 关联停车记录 ID（外键） |
| 3 | `plate_number` | VARCHAR | 20 | NOT NULL | — | 车牌号（冗余，便于直接查询） |
| 4 | `amount` | DECIMAL | (10,2) | NOT NULL | — | 缴费金额（元） |
| 5 | `park_hours` | INT | — | NOT NULL | — | 计费小时数（向上取整） |
| 6 | `status` | ENUM | — | NOT NULL | 'valid' | 状态：valid-有效 / expired-已过期 |
| 7 | `pay_time` | DATETIME | — | NOT NULL | CURRENT_TIMESTAMP | 缴费时间 |

#### status 枚举说明

| 值 | 含义 | 说明 |
|-----|------|------|
| `valid` | 有效 | 缴费成功且在有效期内 |
| `expired` | 已过期 | 超时未出场，缴费失效；保留记录用于审计和抵扣计算 |

#### DDL

```sql
CREATE TABLE `payment_record` (
    `id`                 INT             NOT NULL AUTO_INCREMENT,
    `parking_record_id`  INT             NOT NULL COMMENT '关联停车记录ID',
    `plate_number`       VARCHAR(20)     NOT NULL COMMENT '车牌号',
    `amount`             DECIMAL(10,2)   NOT NULL COMMENT '缴费金额',
    `park_hours`         INT             NOT NULL COMMENT '计费小时数',
    `status`             ENUM('valid','expired') NOT NULL DEFAULT 'valid' COMMENT '状态：valid-有效 expired-已过期',
    `pay_time`           DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '缴费时间',
    PRIMARY KEY (`id`),
    INDEX `idx_parking_record_id` (`parking_record_id`),
    INDEX `idx_plate_number` (`plate_number`),
    INDEX `idx_pay_time` (`pay_time`),
    CONSTRAINT `fk_payment_parking`
        FOREIGN KEY (`parking_record_id`) REFERENCES `parking_record` (`id`)
        ON DELETE CASCADE ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='缴费记录表';
```

#### 索引说明

| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| `PRIMARY` | `id` | 主键 | 唯一标识 |
| `idx_parking_record_id` | `parking_record_id` | 普通索引 | 通过停车记录查缴费 |
| `idx_plate_number` | `plate_number` | 普通索引 | 按车牌查缴费历史 |
| `idx_pay_time` | `pay_time` | 普通索引 | 按时间统计收入 |
| `fk_payment_parking` | `parking_record_id` | 外键 | 确保数据完整性 |

## 五、常用查询 SQL

### 5.1 查询某车牌当前在场记录

```sql
SELECT * FROM parking_record
WHERE plate_number = '豫B·60P70'
  AND status IN ('parking', 'paid')
ORDER BY entry_time DESC
LIMIT 1;
```

### 5.2 计算某车牌的停车费用

```sql
-- 参数：入场时间 entry_time
-- 计费公式：首小时5元，之后每小时3元，向上取整
SELECT 
    plate_number,
    entry_time,
    TIMESTAMPDIFF(HOUR, entry_time, NOW()) AS raw_hours,
    CASE 
        WHEN TIMESTAMPDIFF(MINUTE, entry_time, NOW()) = 0 THEN 1
        WHEN TIMESTAMPDIFF(MINUTE, entry_time, NOW()) % 60 = 0 
            THEN TIMESTAMPDIFF(HOUR, entry_time, NOW())
        ELSE TIMESTAMPDIFF(HOUR, entry_time, NOW()) + 1
    END AS park_hours,
    CASE 
        WHEN TIMESTAMPDIFF(MINUTE, entry_time, NOW()) = 0 THEN 5
        WHEN TIMESTAMPDIFF(MINUTE, entry_time, NOW()) % 60 = 0
            THEN 5 + GREATEST(TIMESTAMPDIFF(HOUR, entry_time, NOW()) - 1, 0) * 3
        ELSE 5 + GREATEST(TIMESTAMPDIFF(HOUR, entry_time, NOW()), 0) * 3
    END AS amount
FROM parking_record
WHERE plate_number = '豫B·60P70'
  AND status = 'parking'
ORDER BY entry_time DESC
LIMIT 1;
```

> **注：** 实际系统中计费逻辑在 Python 后端 `services/billing.py` 中实现，SQL 仅用于手动查询验证。

### 5.3 管理后台 — 停车记录列表

```sql
SELECT 
    r.id,
    r.plate_number,
    r.entry_time,
    r.exit_time,
    r.status,
    IFNULL(p.amount, 0) AS paid_amount
FROM parking_record r
LEFT JOIN payment_record p ON p.parking_record_id = r.id
WHERE 1=1
  AND r.status = 'parking'          -- 可选：筛选在场车辆
  AND r.plate_number LIKE '%豫B%'    -- 可选：按车牌搜索
ORDER BY r.entry_time DESC
LIMIT 20 OFFSET 0;
```

### 5.4 管理后台 — 收费统计

```sql
-- 今日收入（仅有效缴费）
SELECT COALESCE(SUM(amount), 0) AS today_income
FROM payment_record
WHERE DATE(pay_time) = CURDATE()
  AND status = 'valid';

-- 本月收入（仅有效缴费）
SELECT COALESCE(SUM(amount), 0) AS month_income
FROM payment_record
WHERE YEAR(pay_time) = YEAR(CURDATE())
  AND MONTH(pay_time) = MONTH(CURDATE())
  AND status = 'valid';

-- 总收入（仅有效缴费）
SELECT COALESCE(SUM(amount), 0) AS total_income
FROM payment_record
WHERE status = 'valid';

-- 当前在场车辆数
SELECT COUNT(*) AS parked_count
FROM parking_record
WHERE status IN ('parking', 'paid');
```

## 七、数据量估算与扩展考量

| 指标 | 估算 |
|------|------|
| 日均停车量 | 约 100 辆（演示环境） |
| 单条 parking_record | 约 300 字节 |
| 单条 payment_record | 约 120 字节 |
| 年数据量（365天） | 约 15.3 MB |
| 是否需要分表 | 不需要，单表千万级以下无需考虑 |

> 当前为教学演示项目，两张表 + 索引即可满足需求，无需引入缓存或读写分离。
